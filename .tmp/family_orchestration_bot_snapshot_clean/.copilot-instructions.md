# Copilot Instructions — Family Orchestration Bot

## System Purpose
This repository is a Python-based household orchestration backend using FastAPI + SQLite.

The system is event-driven and schema-driven. All logic must conform to defined schemas under `/schemas`.

---

## Current Phase: Phase 1 (Execution Spine Only)

Phase 1 goal is strictly limited to building a working event pipeline:

### Required Capability
- Accept SystemEvent via HTTP POST `/event`
- Validate using Pydantic schema
- Route event by type
- Execute Task module only
- Persist Task in SQLite
- Return created task response

---

## Hard Constraints (Do Not Violate)

### Architecture constraints
- Python only (FastAPI backend)
- SQLite only (no external DBs)
- No AI/LLM integration in Phase 1
- No workflows engine
- No email/calendar ingestion
- No agent framework

### Module constraints
Only the Task module is active in Phase 1.

All other modules must remain stubbed or unused.

---

## Required System Flow

All implementations must follow:

API → SystemEvent → Router → Module Service → SQLite → Response

No additional layers unless explicitly required for Phase 1.

---

## Schema Rules

- All entities must align with `/schemas` definitions
- Do not invent new fields outside schema definitions
- Use metadata fields for extensibility only
- Always include:
  - id
  - household_id
  - timestamps (created/updated where applicable)

---

## Implementation Behavior Rules

When generating or modifying code:

1. Prefer minimal implementations
2. Avoid over-engineering or abstraction layers
3. Implement only what is required for current Phase
4. Do not build future phases early
5. Keep services deterministic (no AI logic)

---
